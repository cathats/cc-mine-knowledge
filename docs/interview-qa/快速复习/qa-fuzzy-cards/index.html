高频题模糊化答案卡片

说明：以下答案刻意保留弹性表达，避免背稿痕迹。建议按“结论一句话 + 场景例子”回答。

React / TS / JS​

1. useState 更新为什么看起来不是立刻生效​

通常会被批量调度，同一轮里可能先合并再提交，所以读取旧值是常见现象。需要基于前值更新时，函数式写法更稳。

2. useEffect 的依赖数组怎么理解​

依赖本质是副作用触发条件。漏依赖可能读到旧闭包，乱加依赖又会导致重复执行，关键是把副作用边界拆清楚。

3. useMemo 和 useCallback 的区别​

前者缓存计算结果，后者缓存函数引用。它们都不是“默认要用”，只有在重渲染成本明显或引用稳定性重要时才有收益。

4. useRef 为什么改了不触发渲染​

它更像跨渲染周期的容器，不参与渲染驱动。适合存放 DOM 引用、定时器句柄、上次值等非视图状态。

5. 什么情况下考虑自定义 Hook​

当同类状态逻辑在多个页面重复出现时，比如筛选+分页+请求+重置，抽成 Hook 能让组件更薄、测试更集中。

6. 父组件更新为什么子组件可能也更新​

父组件重新执行会重新创建子节点描述，子组件默认会走一轮对比流程。想减少无效更新通常要配合 memo 和稳定引用。

7. key 在列表渲染里的核心作用​

帮助框架识别“谁是谁”，减少错误复用和额外销毁重建。业务里更推荐稳定唯一 id，而不是随机值或索引。

8. 全局状态和局部状态怎么取舍​

页面私有交互优先局部状态；跨页面共享或长期保留才考虑全局状态。放全局过多会提高耦合和维护成本。

9. 服务端状态为什么常交给查询库​

服务端状态有缓存、重试、失效、并发等问题，查询库对这些场景更系统，手写很容易漏边界。

10. type 和 interface 怎么选​

日常都能表达对象结构。更看团队约定和扩展方式：例如组合复杂类型时 type 更顺手，声明合并场景里 interface 更常见。

手写题​

11. 防抖和节流差异​

防抖偏“停下来再执行”，节流偏“固定频率执行”。输入框搜索常用防抖，滚动监听常用节流。

12. Promise.all 的关键点​

保持结果顺序、任一失败整体失败、空数组直接完成。实现时通常需要计数器和按索引回填。

13. 深拷贝要关注哪些边界​

循环引用、数组对象混合、特殊对象类型。面试里先讲清基础版本，再说明可扩展策略通常更稳。

14. 发布订阅模式适合什么场景​

解耦事件触发方和响应方，适合跨模块通知。要注意事件泄漏和解绑时机。

浏览器 / 网络 / 性能​

15. 从输入 URL 到渲染页面的主流程​

可概括为：解析地址、网络建连、请求响应、解析资源、构建渲染树、布局与绘制。面试里抓主干，不必陷入过细实现。

16. 强缓存和协商缓存的区别​

前者命中后通常不走内容下载，后者会向服务端确认资源是否变化。一个偏本地直接复用，一个偏条件校验。

17. CORS 的本质​

浏览器层面的同源限制与放行机制。服务端通过响应头声明允许范围，前端常见的是预检与凭证配置问题。

18. 宏任务和微任务怎么理解​

可理解为两种调度队列，微任务一般在当前轮次更早清空。答题时说明常见来源并给一个执行顺序例子就够用。

19. SSE 和 WebSocket 如何选​

持续单向推送常优先 SSE；需要双向高频实时互动更倾向 WebSocket。要结合业务通信模型而不是只看“新不新”。

20. 前端性能优化怎么分层回答​

先首屏（加载链路），再渲染（减少无效更新），再传输（体积与缓存），最后交互（长任务拆分）。分层比堆术语更有说服力。

AI 项目表达（模糊化）​

21. 为什么引入检索增强​

长链路生成里，历史信息会持续累积，纯上下文拼接成本高且不稳定。引入检索是为了在可控成本下提升一致性。

22. 向量检索在项目中的价值​

它解决“语义相近但关键词不同”的召回问题，尤其适合角色设定、章节摘要、世界观片段这类文本资产。

23. 为什么做流式展示​

用户不用等整段完成再看到结果，反馈更及时，也更适合中途打断和调整生成方向。

24. 资源库怎么设计更实用​

通常按实体类型分层，并维护章节与资源的引用关系，方便后续生成时做关联召回与一致性校验。

25. AI 内容系统的工程难点​

难点常在连续生成一致性，不在单次生成。需要处理记忆组织、冲突修正、上下文裁剪和可追溯性。

面试表达与求职执行​

26. 项目讲解的通用结构​

背景、目标、技术方案、关键取舍、最终结果。每段控制在 20～30 秒，便于追问展开。

27. 如何回答“你最大的价值是什么”​

尽量从“解决复杂问题的稳定交付能力”切入，并给出跨角色协作与结果数据支撑。

28. 被问到不会的问题怎么办​

先界定已知边界，再给排查路径和可验证方案。比硬猜答案更能体现工程可信度。

29. 为什么强调边投边补​

市场窗口和反馈周期都很短，先进入真实面试回路，才能用一手反馈提升命中率。

30. 如何避免背稿感​

保留关键词，不背长句；同一问题准备两种说法；用项目细节替代空泛定义。
