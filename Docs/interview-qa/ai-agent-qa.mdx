# AI 未分类面试 Q&A

下面整理成面试表达版本，支持点击“显示答案”查看。

<QAItem question="Q1：什么是 AI Agent？">
AI Agent 可以理解为一个具备自主决策能力的智能系统。它不是简单地回答问题，而是能够根据目标进行推理、调用工具、观察结果，并持续调整下一步动作，最终完成任务。

简单来说：

```text
AI Agent = 大模型 + 工具 + 记忆 + 执行循环
```

比如用户说“帮我订一份晚饭”，Agent 不只是回复推荐菜品，而是可以理解需求、查询餐厅、结合用户偏好、计算价格，甚至完成下单。
</QAItem>

<QAItem question="Q2：AI Agent 的三个核心构成要素是什么？">
AI Agent 通常包含三个核心要素：

1. **LLM，大语言模型**
2. **Tools，工具**
3. **Memory，记忆**
</QAItem>

<QAItem question="Q3：LLM 在 Agent 中起什么作用？">
LLM 是 Agent 的“大脑”，主要负责理解用户意图、分析问题、规划步骤和做出决策。

例如用户说：

```text
我晚上想吃点不太辣的，预算 50 元以内
```

LLM 需要理解出几个关键信息：

```text
时间：晚上
口味：不太辣
预算：50 元以内
任务：推荐或订餐
```

然后判断下一步应该调用餐厅搜索工具，还是先询问用户地址。
</QAItem>

<QAItem question="Q4：Tools 在 Agent 中起什么作用？">
Tools 是 Agent 的“手”，负责执行大模型本身做不到的事情，比如搜索、查数据库、调用接口、下单、发消息等。

比如智能订餐助手可以调用：

```text
餐厅搜索工具
菜单查询工具
价格计算工具
下单工具
支付工具
消息通知工具
```

没有工具的话，Agent 只能“说”；有了工具之后，Agent 才能真正“办事”。
</QAItem>

<QAItem question="Q5：Memory 在 Agent 中起什么作用？">
Memory 是 Agent 的“上下文和经验”，用于保存对话历史、用户偏好、历史任务结果等信息。

比如用户之前说过：

```text
我不吃香菜
我喜欢川菜
公司地址是 XXX
```

下次用户说“帮我点个午饭”，Agent 就可以自动避开香菜，优先推荐川菜，并默认配送到公司地址。
</QAItem>

<QAItem question="Q6：为什么说“大模型不等于 Agent”？">
因为大模型主要负责理解和生成内容，而 Agent 除了会理解和回答，还能调用工具、保存记忆、执行动作，并根据外部结果持续调整策略。

区别可以这样理解：

```text
大模型：负责想和说
Agent：负责想、查、做、记、改
```

例如：

```text
大模型可以告诉你附近可能有哪些餐厅
Agent 可以实际查询外卖平台、筛选餐厅、计算价格，并发起下单
```

所以 Agent 是基于大模型构建的任务执行系统，而不是大模型本身。
</QAItem>

<QAItem question="Q7：如果要构建一个智能订餐助手，会用到哪些工具？">
至少会用到以下几类工具：

### 1. 餐厅 / 菜品搜索工具
用于根据用户位置、口味、预算筛选餐厅和菜品。

### 2. 用户偏好查询工具
用于读取用户历史偏好。

### 3. 菜单 / 库存 / 营业状态查询工具
用于判断餐厅是否营业、菜品是否可售、是否支持配送。

### 4. 价格 / 优惠券计算工具
用于计算总价、配送费、满减和优惠券。

### 5. 下单 / 支付工具
用于真正创建订单或发起支付。

### 6. 消息通知工具
用于提醒用户订单状态。
</QAItem>

<QAItem question="Q8：智能订餐助手的执行流程大概是什么？">
可以分为几个步骤：

```text
1. 理解用户需求
2. 读取用户偏好和地址
3. 搜索符合条件的餐厅和菜品
4. 查询菜单、库存和配送状态
5. 计算价格和优惠
6. 向用户确认
7. 创建订单
8. 通知订单状态
```

这个流程体现了 Agent 的典型工作方式：

```text
理解 → 调工具 → 观察结果 → 再判断 → 再执行
```
</QAItem>

<QAItem question="Q9：什么是 ReAct 模式？">
ReAct 是 Reasoning + Acting 的缩写，意思是“推理 + 行动”。

它让 Agent 不是一次性给答案，而是按照下面的循环执行任务：

```text
Reason：思考下一步做什么
Act：调用工具或执行动作
Observe：观察工具返回结果
Reason：根据结果继续判断
Act：继续执行
Final：输出最终结果
```
</QAItem>

<QAItem question="Q10：LangChain 和 LangGraph 是什么关系？">
LangChain 更像是 LLM 应用开发的组件库，提供模型、Prompt、Tool、Retriever、Agent 等能力。

LangGraph 更像是一个 Agent 编排框架，用来构建复杂、有状态、可控的 Agent 工作流。

可以这样理解：

```text
LangChain：提供能力组件
LangGraph：负责流程编排
```
</QAItem>

<QAItem question="Q11：什么时候用 LangChain，什么时候用 LangGraph？">
如果只是做简单的 RAG、工具调用、问答机器人，用 LangChain 就够。

如果任务流程比较复杂，比如需要多步骤执行、失败重试、人工确认、多 Agent 协作、状态持久化，就更适合用 LangGraph。

可以总结为：

```text
简单 AI 应用：LangChain
复杂 Agent 工作流：LangGraph
```
</QAItem>

<QAItem question="Q12：如何判断 LangChain 代码是 v0.x 还是 v1.x？">
可以看它使用的 API。

如果看到下面这种写法：

```ts
const executor = await initializeAgentExecutorWithOptions(tools, llm, {
  agentType: 'zero-shot-react-description',
});
```

一般是 **v0.x 旧写法**。
</QAItem>

<QAItem question="Q13：LangChain v1.x 推荐的 Agent 写法是什么？">
v1.x 更推荐使用 `createAgent` 这类新 API。

```ts
import {createAgent} from 'langchain';

const agent = createAgent({
  model,
  tools,
  systemPrompt: 'You are a helpful assistant.',
});
```

可以这样记：

```text
v0.x：initializeAgentExecutorWithOptions
v1.x：createAgent
```
</QAItem>

<QAItem question="Q14：面试中如何一句话总结 AI Agent？">
```text
AI Agent 是基于大语言模型构建的任务执行系统，它通过 LLM 进行理解和决策，通过 Tools 与外部系统交互，通过 Memory 保存上下文和经验，并在推理、行动、观察的循环中完成复杂任务。
```
</QAItem>

<QAItem question="Q15：面试中如何一句话总结 LangChain 和 LangGraph？">
```text
LangChain 主要提供构建 LLM 应用所需的模型、工具、检索器等组件；LangGraph 则更关注复杂 Agent 的状态管理和流程编排，适合多步骤、多分支、可恢复的 Agent 工作流。
```
</QAItem>

<QAItem question="Q16：这类问题的高分表达模板">
面试时可以按这个结构回答：

```text
我理解 AI Agent 不是单纯的大模型，而是一个围绕目标自动执行任务的系统。

它通常由三部分组成：
第一是 LLM，负责理解、推理和决策；
第二是 Tools，负责调用外部能力，比如搜索、查库、下单；
第三是 Memory，负责保存上下文、用户偏好和历史经验。

以智能订餐助手为例，LLM 会理解用户想吃什么，Memory 会记住用户不吃香菜、喜欢川菜，Tools 会负责查询餐厅、计算价格、创建订单。整个过程通常会以 ReAct 的方式运行，也就是边推理、边行动、边观察结果，直到完成任务。
```
</QAItem>
