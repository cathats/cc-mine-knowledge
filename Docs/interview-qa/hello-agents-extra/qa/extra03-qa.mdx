# Extra03 Dify 智能体搭建 Q&A

对应原文：[Extra03-Dify智能体创建保姆级操作流程](https://github.com/datawhalechina/hello-agents/blob/main/Extra-Chapter/Extra03-Dify%E6%99%BA%E8%83%BD%E4%BD%93%E5%88%9B%E5%BB%BA%E4%BF%9D%E5%A7%86%E7%BA%A7%E6%93%8D%E4%BD%9C%E6%B5%81%E7%A8%8B.md)

<QAItem question="Q1：Dify 从零搭 Agent 的推荐路径是什么？">
先准备插件和 MCP，再搭 Chatflow 骨架，然后用问题分类器分流到不同能力模块，最后接入外部工具并迭代提示词。
</QAItem>

<QAItem question="Q2：为什么要先做问题分类器？">
分类器能把不同任务路由到专门子流程，减少单提示词负担，提高响应稳定性与可维护性。
</QAItem>

<QAItem question="Q3：角色提示词设计常见结构？">
通常用 Role / Background / Task / Limit / Output Example 结构，明确边界、目标和输出格式，降低歧义。
</QAItem>
