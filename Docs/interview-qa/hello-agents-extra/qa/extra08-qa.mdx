# Extra08 如何写好 Skill Q&A

对应原文：[Extra08-如何写出好的Skill](https://github.com/datawhalechina/hello-agents/blob/main/Extra-Chapter/Extra08-%E5%A6%82%E4%BD%95%E5%86%99%E5%87%BA%E5%A5%BD%E7%9A%84Skill.md)

<QAItem question="Q1：Skill 的最小形态与完整结构分别是什么？">
最小形态是可触发的任务说明；完整结构包含触发元数据、核心指令、脚本/参考/资产等分层资源。
</QAItem>

<QAItem question="Q2：为什么强调“简洁”是根本约束？">
Skill 不是文档百科，越冗长越会干扰模型决策；应把高频关键规则写清，把细节放到按需加载资源中。
</QAItem>

<QAItem question="Q3：Skill 自由度如何控制？">
按任务风险分档：高风险流程低自由度（脚本化/强约束），探索型任务中自由度，创意型任务高自由度。
</QAItem>
